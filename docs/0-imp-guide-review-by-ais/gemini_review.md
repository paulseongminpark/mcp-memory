# mcp-memory 온톨로지 설계 비판적 검토

작성하신 mcp-memory 온톨로지 설계 문서를 깊이 있게 검토했습니다. 뇌과학의 헤비안 학습, BCM 규칙, 가지치기 등의 기전을 소프트웨어 아키텍처로 치환한 논리가 매우 탄탄합니다. 스스로 진화하는 지식 온톨로지를 구축하려는 목적에 정확히 부합하는 설계입니다.

다만, 이 시스템이 실제로 스케일업되고 장기적으로 운영될 때 발생할 수 있는 구조적 마찰과 병목들이 보입니다. 비판적 관점에서 4가지 핵심 설계 이슈를 정리했습니다.

## 1. 50x48x6 분류 체계와 유기적 연결의 충돌
설계의 본질은 뉴런처럼 유연하게 연결되는 네트워크(리좀적 구조)를 지향하고 있습니다. 동시에 50개의 노드 타입, 48개의 관계 타입, 6단계의 엄격한 수직적 레이어(L0~L5)를 규정하고 있습니다. 

이는 유기적인 시스템에 매우 관료적인 행정 절차를 도입한 형태입니다. 모든 기억이 반드시 이 촘촘한 격자망 안에 완벽히 분류되어야만 작동한다면, 시스템은 유연성을 잃고 분류 작업 자체에 에너지를 소모하게 됩니다. 특정 관찰(L0)이 중간 단계를 거치지 않고 곧바로 철학(L3)이나 핵심 가치관(L4)과 강하게 결합하는 식의 '비선형적 도약'이 현재의 승격(Promotion) 게이트 구조에서는 차단될 확률이 높습니다.

## 2. Drift 탐지의 부작용 (환각 차단 vs 패러다임 시프트 억제)
Enrichment 과정에서 코사인 유사도가 0.5 미만일 때 업데이트를 차단하는 것은 거짓 기억(False memory)의 증폭을 막는 훌륭한 방어 기제입니다. 

하지만 이 엄격한 차단은 **'근본적인 깨달음(Epiphany)' 마저 오류로 취급하여 버리게 만듭니다.** 과거의 관찰이나 패턴이 새로운 맥락을 만나 완전히 다른 의미로 재해석되는 패러다임 시프트가 발생할 때, 임베딩 벡터의 위치는 필연적으로 크게 이동합니다. 시스템은 이를 새로운 맥락과 의미의 진화로 받아들이지 못하고, 데이터 오염으로 판정해 차단할 확률이 높습니다. 완전 차단 대신 'Human Review(인간 검토) 큐'로 넘기는 우회로가 필요합니다.

## 3. Recall 파이프라인의 연산 부하 집중
Recall 단계에서 벡터 검색, 텍스트 검색, 그래프 탐색을 동시 수행하고, 그 결과를 바탕으로 BCM 학습(연결 강화/약화)을 계산하며, UCB를 적용해 탐색 보너스까지 부여합니다. 

현재 3,230개의 노드에서는 문제가 없겠지만, 노드가 수만 개로 늘어나고 엣지가 복잡해지면 이 파이프라인은 심각한 응답 지연(Latency)을 유발합니다. 인간의 뇌도 기억을 떠올리는 즉시 시냅스를 재조정하지 않습니다. 검색(Fetch)과 학습(Weight Update)을 분리하여, BCM 계산과 엣지 가중치 업데이트는 비동기 백그라운드 작업이나 수면(Daily Enrich) 단계로 미루는 비동기 설계가 안전합니다.

## 4. 시간 기반 가지치기(Pruning)의 맹점
연결 강도가 0.05 미만이고 30일 이상 안 쓰인 엣지를 아카이브하는 규칙은 상당히 공격적입니다. 생물학적 뇌에서 어떤 기억의 시냅스는 몇 년간 전혀 사용되지 않다가도 특정한 트리거를 만나면 즉시 활성화됩니다. 

일괄적인 30일 타이머는 롱테일(Long-tail) 인사이트를 영구적으로 잘라낼 위험이 있습니다. 레이어별로 가지치기 반감기를 다르게 설정해야 합니다. 일회성 관찰(L0)은 30일 만에 쳐내더라도, 프레임워크(L2) 이상의 연결은 훨씬 더 긴 유예 기간을 주거나, 강도가 0에 수렴하더라도 완전히 끊지 않고 미세한 흔적을 남겨두는 방식이 유리합니다.
