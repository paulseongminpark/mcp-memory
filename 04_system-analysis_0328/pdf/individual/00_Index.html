Section 01

구조

삼부작 + 방법론 + 5개 렌즈의 3층 아키텍처

9개 문서는 세 개의 층으로 조직된다.

삼부작은 뼈대다. Alpha(WHY)는 연역적으로 구조의 필연성을 논증한다. Beta(WHAT)는 귀납적으로 4,786개 노드의 운영 결과를 보고한다. Omega(SO WHAT)는 귀추적으로 새로운 이론을 제안한다.

Praxis는 다리다. 이론과 발견을 실천으로 전환하는 방법론을 제공한다.

5개 렌즈는 살이다. 동일한 시스템을 인지과학, 철학, 소프트웨어 아키텍처, 경쟁 지형, 언어학의 관점에서 각각 조명한다.

Alpha — 왜 구조인가

이론에서 시스템으로의 연역적 논증 · "LLM 시대에 인간의 사고는 어떤 외부 구조를 필요로 하는가?"
#Thesis핵심 주장
0LLM의 구조적 불가능할루시네이션·맥락소멸·비구조성은 스케일링으로 해결 안 된다
1세 가지 간극토큰-의미, 문장-구조, 컨텍스트-세계
2온톨로지의 필연벡터 DB의 세 가지 실패 모드 → 하이브리드 검색
3정적 온톨로지의 한계BFO/DOLCE/Gene Ontology가 개인 인지에 안 맞는 이유
4승격 메커니즘L0→L5 증류 피라미드, BCM 게이트, DB 분포 실증
5역방향 흐름원칙→경험 재해석, WIM 7렌즈, 들뢰즈 반-현실화
6자기교정Correction recall 최우선, 2차 피드백 루프, node #5353/#5369
7세션 주입작업기억 대응, L2+ 우선, Context as Currency
8에코시스템12개 프로젝트, foundation/ 자기참조, 인지 순환
9경쟁 지형2×2 매트릭스, 우상단 빈 공간
10선언LLM은 매체, 매체에는 구조, 구조는 살아있어야

Beta — 무엇이 드러났는가

시스템에서 발견으로의 귀납적 보고 · "4,786개 노드를 운영하면서 무엇을 발견했는가?"
#Finding핵심 발견
0의도 vs 결과메모리 서버 → 메타인지 운영체계
1판단이 가장 많다Decision 987개 = 전체 20.6%. knowledge base가 아니라 decision engine
2실패의 가치Failure 141개. 같은 실패 방지 + 패턴 발견 + Correction 생성
3타입은 사후 발견materialized 25/51. Correction 사후 추가. L4–L5 아직 비어있음
4Context as Currency더 넣으면 더 나쁘다. 에이전트 24→15, 스킬 14→9 + 4단계
5프로토콜의 필요Claude 단정 패턴 → Phase v3, PDR, foundation 3문서
6멀티에이전트 카오스동시 쓰기 충돌 → .ctx/, rulesync, worktree
7시스템 도입 전후 비교대조군 보완 — 도입 전후 정성적 비교
8측정이 메타인지auto-iterate L1–L5 분포 → 사고 편향 가시화
9온톨로지는 거울DB 분포 = 만든 사람의 사고 구조
10구현이 이해를 바꾼다이론 읽기 ≠ 이론 구현. 들뢰즈·BCM·CLS의 코드화

#Proposition핵심 제안
0이론의 끊김들뢰즈·CLS·BCM 각각의 설명 범위와 한계
1인지적 공동 구성Human-AI = 도구 사용이 아니라 co-construction
2Extended Cognition외부 기억 = 기억이 아니라 사고의 골격
3인식론적 온톨로지존재론이 아니라 "앎의 방식"을 구조화
4인지적 증류승격 = 기존 학습 이론에 없는 새 모델
5실패 = 최고 순도 지식포퍼 반증주의의 개인 인지 적용
6Context = 제4 자원CPU·Memory·Storage + Context
7프로토콜 > 프롬프트인지적 프로토콜 디자인이라는 새 영역
8불안정 = 생명괴델 불완전성 vs 루만 자기생산
9설계 원리 7가지구조 먼저 / 승격 / 역방향 / 실패 1급 / 화폐 / 프로토콜 / 진화
10내가 구조를 만들고...삼부작 자체가 승격 메커니즘과 동형

#Phase핵심 행동
0전제지속적 LLM 협업 + 자기관찰 의지 + 구조화 감각

4첫 100개 노드판단 있는 것만. 실패 적극 기록
5승격 경로수동→반자동. confidence + 참조횟수 기반 임계값
6세션 주입L2+ 우선 + 현장성 보조 + Correction 최우선

8역방향 흐름Principle 20개+ → 해석 렌즈 → 새 입력 필터링
9측정과 반복레이어 분포, Failure 비율, 승격률
10함정과잉 설계, 분류 강박, 도구 의존. 시스템 ≠ 목적
